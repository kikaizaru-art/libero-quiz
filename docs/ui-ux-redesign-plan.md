# UI/UX 再構成プラン

最低限の機能が揃った現状(v1)をベースに、「1日5問の習慣化アプリ」としての体験を最優先に構成を見直すプラン。フェーズ順に実装できるよう、課題 → 方針 → 具体案 → 実装順の構成でまとめる。

## 現状の構成と課題

現状は7画面(ホーム/分野一覧/ステージ選択/クイズ/リザルト/記録/実績)+下部ナビ4タブ。機能は揃っているが、体験面で以下の課題がある。

### 1. 「今日の学習を始める」までが遠い(最重要)

ホーム →「学習を始める」→ 分野一覧 → ステージ選択 → クイズ開始、と**最初の問題まで3タップ**必要。毎日使う習慣化アプリでは、起動 → 学習開始が1タップであるべき。「どの分野のどのステージをやるか」を毎回ユーザーに選ばせるのは、意思決定コストが高く継続の妨げになる。

### 2. ホームが「ダッシュボード」になっていて「行動の起点」になっていない

ホームの並びは「XPヘッダー → サブタイトル → 本日の目標 → 開始ボタン → 復習 → 学習記録」。主役であるべき開始ボタンが目標カードの下に埋もれている。また「学習記録」カードは記録タブとほぼ同じ内容の重複で、ホームを縦に長くしているだけ。

### 3. 導線の重複と迷い

「学習を始める」ボタンと下部ナビの「学習」タブが同じ分野一覧に行く二重導線。役割が分かれていないため、ホームのボタンの存在意義が弱い。

### 4. クイズ中の体験の粗さ

- 解答後、解説が選択肢の**下に**出現するため、問題文が長いとスクロールしないと解説と「次へ」が見えない
- 中断確認がネイティブの `confirm()` でトーンが崩れる
- 正解判定が `textContent` の一致比較(`app.js` の `answer()`)なので、同じ文字列の選択肢があると誤動作するバグ含み

### 5. リザルトで「学び」が完結しない

リザルトはスコアとXPのみ。どの問題を間違えたかのふりかえりがなく、「間違えた問題が復習リストに入った」ことも伝わらない(復習機能の存在に気づきにくい)。

### 6. 「実績」タブの格が高すぎる

実績は閲覧頻度の低い画面なのに、ナビの1/4を占有。一方で継続の要である「復習」はホームのカード内にしか導線がない。

---

## 再構成の方針

> **「開いたら1タップで今日の5問。終わったら成果が見えて、明日も開きたくなる」**

1. **開始までの摩擦を最小化** — ホームの主CTAを「今日の5問」に。分野選択はやりたい人だけの経路にする
2. **ホームは行動の起点に特化** — 見せるのは「今日やること」と「継続の実感(ストリーク)」だけ。詳細な統計は記録タブへ
3. **1画面1目的** — 重複コンテンツを排除し、各画面の役割を明確化
4. **学習ループを閉じる** — 出題 → 解説 → ふりかえり → 復習、を体験として繋げる

## 新しい画面構成

### ナビゲーション(4タブ → 4タブ、役割変更)

| 現行 | 新構成 | 変更点 |
|---|---|---|
| ホーム | **ホーム** | 「今日やること」に特化 |
| 学習 | **学習** | 分野一覧+「続きから」導線を強化 |
| 記録 | **記録** | サマリー/カレンダー/分野別 + **実績をセグメント統合** |
| 実績 | **復習** | 実績を記録に吸収し、空いた枠を復習に |

復習は「間違いを克服する」という継続の中核機能なので、タブに昇格させて常時見えるようにする。復習タブには待ち問題数のバッジ(例: `復習 ③`)を表示。

### ホーム(全面改修)

```
┌──────────────────────────┐
│ Lv.3 ▓▓▓▓░░ 120/200XP  連続4日 │ ← 現状維持(1行に圧縮)
├──────────────────────────┤
│  今日の5問                     │
│  「哲学 ステージ2」の続きから     │ ← 何が出るか予告
│  [ 始める ]                    │ ← 主CTA。1タップで即開始
├──────────────────────────┤
│ 本日の目標  ○●●  2/3達成       │ ← 3行 → 1行サマリーに圧縮
│                (タップで展開)   │
├──────────────────────────┤
│ 復習待ちが3問あります [復習する] │ ← 復習0件なら非表示
└──────────────────────────┘
```

- **「今日の5問」**: 未クリアの最初のステージを自動選択して即開始(前回の続き優先)。全ステージ星3なら復習 or ランダム5問出題にフォールバック
- **学習記録カードは削除**(記録タブへ一本化)
- **本日の目標は1行サマリー+タップ展開**。3つのミッションを常時全表示しない
- 復習カードは**復習待ちがあるときだけ**表示(空のカードで「ありません」と言わない)

### 学習タブ(分野一覧+ステージ)

- 各分野カードに**進捗リング**(星の獲得率)と「**続きから**」ボタンを追加し、ステージ選択画面を経由せず次の未クリアステージへ直行できるようにする
- ステージ選択画面は「星3を取り直したい」など明示的にステージを選びたい人向けに現状維持
- ロック中ステージの表現は現状の「－/前のステージをクリアで解放」で良い

### クイズ画面

- **解説をボトムシート化**: 解答後、画面下から解説パネルがスライドイン。選択肢の正誤表示はそのまま背後に見え、スクロール不要で「次へ」が常に親指の位置に来る
- **中断確認をアプリ内モーダルに**: `confirm()` を廃止し、トーンの揃ったダイアログに(「進行中の記録は保存されません」の文言は維持)
- **正解判定をインデックス比較に修正**(バグ修正): ボタンに `data-index` を持たせ、`textContent` 比較をやめる
- 進捗バーに「n / 5」のテキストを併記(バーだけだと残り問数が読めない)

### リザルト画面

- スコア・星・XPの下に「**ふりかえり**」セクションを追加: 間違えた問題の一覧(問題文+正解)を表示し、「復習リストに追加しました」を明示
- これにより「間違い → 復習」のループが体験として繋がり、復習機能の発見性が上がる

### 記録タブ

- 上部にセグメントコントロール `[ 学習記録 | 実績 ]` を置き、実績画面を統合
- 学習記録側は現状の構成(サマリー/カレンダー/分野別正答率)を維持

### 復習タブ(新設)

- ホームの復習カードの内容を画面として独立
- 復習待ち問題数、分野ごとの内訳、間違えた回数の多い順リスト(問題文の冒頭のみ表示、ネタバレ防止で正解は出さない)
- 「5問復習する」CTA

## デザインシステムの整備(横断)

現状の `style.css` はダークテーマ固定・ad hoc な値が多い。再構成に合わせて以下を整える。

1. **トークンの体系化**: spacing(4/8/12/16/24)、フォントサイズ(12/13/15/17/22)、既存の色変数に `--surface-2` 等の階層を追加
2. **ライトモード対応**: `prefers-color-scheme` でトークンを切替(通勤中の屋外利用を想定すると効果大)
3. **モーション配慮**: `prefers-reduced-motion` でシェイク・スライド等を無効化
4. **アクセシビリティ**: トーストに `aria-live="polite"`、画面遷移時のフォーカス管理、選択肢ボタンへの `:focus-visible` スタイル
5. **PWA化**(README の今後のアイデアと合致): `manifest.json` + 最小限の Service Worker でホーム画面追加・オフライン動作。習慣化アプリとして「アプリのように開ける」ことの価値が大きい

## コード構成の見直し(任意・低優先)

`app.js` が750行の単一IIFEになっており、画面追加のたびに肥大化する。ビルド不要の制約は保ったまま、ES Modules で分割できる:

```
js/
  data.js        # 現状維持
  state.js       # セーブデータ・XP/レベル/ストリーク計算
  quiz.js        # 出題ロジック(stage/review/daily の3モード)
  screens/       # 画面ごとの描画
  main.js        # 画面遷移とイベント配線
```

`<script type="module" src="js/main.js">` に変えるだけで、ビルドツールなしで動く(`file://` 直開きは不可になるため、README の起動手順をローカルサーバー前提に統一する)。

## 実装フェーズ(推奨順)

| フェーズ | 内容 | 規模 |
|---|---|---|
| **1. コアループ短縮** | 「今日の5問」CTA、ホーム再構成(記録カード削除・目標1行化・復習カード条件表示) | 中 |
| **2. クイズ体験** | 解説ボトムシート、中断モーダル、正解判定バグ修正、進捗テキスト | 中 |
| **3. ナビ再編** | 復習タブ新設、実績を記録にセグメント統合、リザルトにふりかえり追加 | 中 |
| **4. 磨き込み** | ライトモード、reduced-motion、a11y、PWA化 | 中〜大 |
| **5. コード分割** | ES Modules 化(機能追加が続くなら) | 小〜中 |

フェーズ1と2だけでも「開始までの摩擦」と「クイズ中の引っかかり」という二大課題が解消され、日次利用の体験が大きく変わる。3以降は独立して進められる。
