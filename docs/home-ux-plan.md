# ホーム画面 UX 改善プラン(home-ux)

> **進捗**: Phase A〜C まですべて実装済み。実測はホーム全体で
> 新規 556px / 進行中 794px / 日課完了後 673px(いずれも1画面内)。
> 状態帯は 160px(旧: ヘッダー+週+目標で 192px)。
> ※ 統合カードの名称は「つづける」だと未挑戦の実力判定テストと合わないため、
> 実装では「学習メニュー」に変更した(本文中の「つづける」は読み替え)。

> 課題意識:機能が増えた結果、ホームが「同格カードの縦積みダッシュボード」に
> 戻ってしまった。ui-ux-redesign-plan の「ホームは行動の起点に特化」という方針を、
> 現在の機能数(7カード)を前提に立て直す。

## 現状の実測(2026-07 時点)

進行中ユーザー(ストリークあり・復習待ちあり・実践解放済み)のホームは
**縦 約1600px = 2画面分**。構成は以下の通り(上から)。

| # | カード | 高さ | 性質 |
|---|---|---|---|
| 1 | ヘッダー(Lv/XP/ストリーク) | 81px | 状態 |
| 2 | 今週の学習ドット | 62px | 状態 |
| 3 | 今日の5問 | 202px | 行動(主役) |
| 4 | 学習を進める | 145px | 行動 |
| 5 | 実力判定テスト | 164px | 行動(低頻度) |
| 6 | 実践問題 | 166px | 行動 |
| 7 | 本日の目標(3行フル表示) | 193px | 状態 |
| 8 | 復習 | 166px | 行動 |
| 9 | 今日の一語り | 231px | 読み物 |

### 何が問題か

1. **選択肢過多で「次の1タップ」が決まらない。** 主役以外に6枚のカードが
   「タイトル+説明文+同じ見た目のグレーボタン」で並び、視覚的にすべて等価。
   毎日開く画面で、毎回「どれを押すべきか」を考えさせている。
2. **性質の違うものが同列。** 「学習を進める/復習/実践問題」は行動、
   「本日の目標」は状態表示、「今日の一語り」は読み物。行動・状態・読み物が
   混ざって縦に積まれているため、画面に物語(上から読む順序)がない。
3. **実力判定テストの格が高すぎる。** 月に数回の腕試し機能が3番目の特等席を
   常時占有し、毎回同じ2行の説明文を表示している。
4. **本日の目標が3行フル表示。** ui-ux-redesign-plan で「1行サマリー+タップ展開」
   と決めた項目が未実装のまま残っている。
5. **今日の一語りが最下部。** knowledge-to-action の日次タッチポイントとして
   作った機能が、2画面スクロールしないと見えない位置にある。
6. **同文言の主ボタンが並ぶ。** 新規ユーザーでは「始める」(今日の5問)の直下に
   「始める」(学習を進める)が並び、区別がつかない。

## 方針 — ホームを3ゾーンの物語にする

> 上から「**今日の状態 → 今日やること → 今日のおまけ**」の順に1画面で読み切れること。
> 行動ボタンは「主役1つ+リスト行」。カード数 7 → 4、縦 1600px → 900px 前後を目標。

```
┌────────────────────────────┐
│ Lv.3 ▓▓▓░ 70/200XP   連続6日     │ ← ゾーン1: 状態
│ 日月火水木金土  ・ 本日の目標 1/3    │   (ヘッダー+週+目標を帯に圧縮)
├────────────────────────────┤
│ 今日の5問                        │ ← ゾーン2: 今日やること
│ 音楽・科学・歴史・哲学・芸術         │
│ [ 始める ]            ← 唯一の主CTA │
├────────────────────────────┤
│ つづける                         │
│  復習                2問       › │ ← 行動は1枚のリスト行に統合
│  哲学 中級の続き              › │   (説明文なし・件数バッジ+シェブロン)
│  実践問題            5問       › │
│  実力判定テスト   ベストB      › │
├────────────────────────────┤
│ 今日の一語り                      │ ← ゾーン3: 読み物(コンパクト)
│ [芸術] 葛飾北斎 — 海外ゲストとの…  │   (タイトル+冒頭1行、タップで全文)
└────────────────────────────┘
```

## Phase A — 並び替えと圧縮(低コスト・まずここまで)

構造は変えず、順序・文言・表示量だけ直す。

### A-1. 本日の目標を1行サマリー+タップ展開に

`missions-card` を「本日の目標 1/3 達成」の1行(+進捗ドット3つ)に畳み、
タップで現在の3行表示を展開。既存マークアップの `hidden` 切替だけで実装できる。
193px → 約50px。

### A-2. 今日の一語りを「今日の5問」の直下へ+コンパクト化

- 位置:主CTAの直下(学習を進めるの上)。「クイズを始める気分ではない日」の
  受け皿として、主CTAの次に目に入る位置に置く
- 表示:分野ピル+タイトル+使いどころの**冒頭1行(ellipsis)**に圧縮し、
  タップでカード全体がライブラリ詳細を開く(「カードを開く」ボタンは廃止して
  カード全体をタップ領域に)。231px → 約90px

### A-3. 実力判定テストの降格

- 説明文2行を削除し、タイトル+ベスト評価ピル+ボタンの1行構成に縮小
- 位置は「つづける」群の最後尾へ(Phase B でリスト行になる前提の暫定)

### A-4. ボタン文言の重複解消

「学習を進める」のボタンを「始める」→「続きから」に統一(新規時は
「哲学 初級から」のように行き先を入れる)。主CTA「始める」はホームで唯一にする。

## Phase B — 「つづける」統合カード(本命)

「学習を進める/復習/実践問題/実力判定テスト」の4カード(計640px)を、
1枚の**アクションリストカード**に統合する。

- 行の構成:`ラベル+補足(件数・続き先・ベスト)+シェブロン`。説明文は行に持たせない
- 行の表示条件と並び順(優先度順):
  1. **復習**(待ちがあるときだけ・件数バッジ)— 最優先。溜まっているものから
  2. **続きのステージ**(常時・「哲学 中級 2/8」のように現在地を補足)
  3. **実践問題**(解放済みがあるときだけ・挑戦可能数)
  4. **実力判定テスト**(常時・ベスト評価を補足)
- ライブラリ相当の行スタイル(`library-item`)を流用できるため、CSSの新規は最小
- ナビの復習バッジ・復習タブは現状維持(導線の重複はあるが、タブは「あとで
  まとめてやる」用、ホーム行は「今すぐ1回やる」用と役割が分かれる)

これでホームは「状態帯 / 今日の5問 / 一語り / つづける」の4ブロックになる。

## Phase C — 状態帯の統合と適応(磨き込み)

### C-1. ヘッダー・週ドット・目標サマリーを1つの「状態帯」に

ヘッダー(81px)+週ドット(62px)+目標1行(50px)を、1枚の帯(約110px)に統合。
「連続6日 ・ 今日の目標 1/3」のように、今日の状態が1視線で読めるようにする。

### C-2. 日課の進行に合わせた入れ替え

- **今日の5問が未完了**:現行どおり主CTAが最上位
- **完了後**:主CTAカードを「完了 ・ 明日は◯◯から出題」の小さな帯に畳み、
  「つづける」リストと「今日の一語り」を上に繰り上げる(もう一歩やりたい人と
  読み物で余韻を残したい人の両方を受ける)
- 既存の `today-hook`(明日の予告)はこの帯に吸収

### C-3. 視覚ヒエラルキーの規約化

- 主CTA(btn-primary)はホームに常に1つだけ
- 行動はリスト行、状態はテキスト/ドット、読み物はカード、と型を固定
- カード間マージンを詰め、進行中ユーザーで**1画面+α(約900px)**に収める

## 実装上の注意

- 並び替えは index.html の DOM 順変更+CSS で完結させ、既存の render 関数の
  ID 参照を壊さない(`home-*-card` の ID は維持)
- 「つづける」統合で削るのはカードの器だけ。startReview / startPractice /
  startExam / pickLearnTarget などのロジックはそのまま行の onclick に付け替える
- 目標の展開状態・一語りの既読などの UI 状態は保存不要(セッション内でよい)。
  保存する場合も学習データ(state)ではなく settings 側に置く
- UI文言に絵文字は使わない。css/index.html のみの変更でも sw.js の CACHE を上げる

## 優先順位の根拠

| Phase | コスト | 効果 |
|---|---|---|
| A 並び替えと圧縮 | 小 | 縦2画面→1.3画面。一語りの発見性が回復 |
| B つづける統合 | 中 | カード7→4。「次の1タップ」が迷わなくなる(本命) |
| C 状態帯と適応 | 中 | 1画面に収まり、完了後の体験まで設計される |

A→B は連続で実施してよい(Bの完成形を前提にAの並び順を決めてある)。
C は A・B の見た目が固まってから着手する。
