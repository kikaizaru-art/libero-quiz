# 毎日続けたくなる動機づけ・習慣化プラン

> **進捗**: Phase 1(1-1〜1-4)、2-1 ストリークフリーズ、4-1 復習の間隔反復
> (SRS: 正解ごとに1→3→7日後へ延期、全区間クリアの次の正解で克服)は実装済み。
> 次の候補は 2-2 ラッキー問題 / 2-3 ウィークリーリキャップ → Phase 3 リマインダー導線。

現状の分析と、それを踏まえた4フェーズの改善計画。設計の軸は習慣ループ
(**きっかけ → 行動 → 報酬**)と、行動科学でよく効く3つの仕掛け
(**損失回避 / 進捗の可視化 / 変動報酬**)。

## 現状の強みと穴

すでにある継続の仕組み:

- 今日の5問(日替わり・初回ボーナス)/ デイリーミッション3種
- XP・レベル / ストリーク(3日・7日バッジ) / 実績16種
- 復習リスト / 月別学習カレンダー / PWAオフライン対応

習慣化の観点で足りないもの:

| 穴 | 症状 |
|---|---|
| 「今日まだやってない」が目立たない | ホームを開いてもストリークが危険かどうか分からない |
| ストリークが壊れやすい | 1日休むと即リセット。長く続けた人ほど失望が大きく、離脱の最大要因 |
| 報酬が固定的 | XPが毎回同じで予測可能。開ける楽しみ(変動報酬)がない |
| 完了後の「明日へのフック」がない | 今日の5問を終えたら、翌日開く理由が提示されない |
| ストリーク報酬が7日で頭打ち | 8日目以降は続けても何も起こらない |
| きっかけ(リマインダー)が皆無 | 思い出す手段がアプリの外にない |
| 振り返りの単位が日だけ | 週単位の達成感・成長実感がない |
| コンテンツ有限(90問) | 全ステージ星3後の「日課の受け皿」が今日の5問だけ |

## Phase 1 — 毎日戻ってくる理由を強くする(最優先・小さく効く)

### 1-1. ホームに「今週の学習」ドット表示

月〜日の7つのドットをホームのストリーク表示の下に置き、学習した日を点灯。
「鎖を切りたくない」心理をホームを開いた瞬間に働かせる。

- データは既存の `state.activity`(YYYY-MM-DD → 解答数)で足りる。追加フィールド不要
- 今日のドットは未学習なら中抜き+強調で「まだ」を示す

### 1-2. ストリークの危機/点火演出

- 今日未学習: ホームのストリーク表示をグレー(消えた炎)にし「今日の5問でつなごう」と添える
- 今日学習済み: 通常色で点灯
- その日最初のクイズ完了時、リザルトに「🔥 連続◯日目!」を大きく表示

`touchStreak()` 済みかどうかは `state.streak.last === todayStr()` で判定できるため、
状態追加なしで実装可能。

### 1-3. リザルトに「明日の予告」

今日の5問の出題は日付シードで決まるため、**明日の分野を先読みできる**。
今日の5問クリア後のリザルトに「明日は 哲学・芸術・… から出題」と予告を出し、
翌日開く理由(ツァイガルニク効果)を作る。

- `dailyItems()` のシードを `daily-${date}` の引数化だけで対応可

### 1-4. ストリーク・継続系バッジの拡充

- 連続: 14日 / 30日 / 50日 / 100日
- 累計学習日数(`Object.keys(state.activity).length`): 10日 / 30日 / 100日
- 「今日の5問を7回クリア」「今日の5問を30回クリア」(要 `totals.dailyClears` カウンタ追加)

## Phase 2 — 損失回避と変動報酬(習慣ループの強化)

### 2-1. ストリークフリーズ(連続記録の保護)★目玉

1日休んでも連続記録が消えない保険。Duolingoで最も効果が実証されている仕組み。

- 7日連続するごとに1個獲得、最大2個ストック
- 1日空いた状態で戻ってきたら自動消費し「フリーズが連続記録を守りました」とトースト
- 2日以上空いたらリセット(フリーズは1日分のみ埋める)
- state追加: `streak.freezes`(所持数)、`touchStreak()` に消費ロジック追加
- ホームのストリーク横に ❄️×n で所持数を表示

### 2-2. ラッキー問題(変動報酬)

今日の5問のうち1問が日替わりでXP2倍。正解時に初めて「⭐ ラッキー問題! XP2倍」と分かる。

- どの問題かは日付シードで決定(`seededRandom` を流用)。状態追加不要
- 「開けてみるまで分からない」予測不能な報酬が、開くこと自体を楽しみにする

### 2-3. ウィークリーリキャップ

週が変わって最初の起動時に、先週のまとめカードを1枚表示:

- 学習日数 / 解答数 / 正解数 / 克服数 / 獲得XP
- 前週との比較(「前の週より +12問」)
- state追加: `lastRecapWeek`(表示済み週のキー)。集計は `state.activity` から算出

## Phase 3 — きっかけの外部化(リマインダー)

サーバーレス静的PWAという制約上、Web Push(pushサーバー必須)や
Notification Triggers API(廃止)は使えない。現実的な選択肢:

### 3-1. カレンダー登録リンク(推奨・確実)

設定画面に「毎日のリマインダー」を追加し、時刻を選ぶと:

- Googleカレンダーの繰り返し予定作成リンク(URLパラメータのみで生成可能)
- `.ics` ファイルのダウンロード(`RRULE:FREQ=DAILY`。iOS/Androidの標準カレンダーで開ける)

サーバー不要・全プラットフォーム対応で最も確実。

### 3-2. PWAインストール誘導の強化

ホーム画面アイコンは最強の視覚的リマインダー。`beforeinstallprompt` を拾って
未インストール時にホームへさりげなく「ホーム画面に追加」カードを出す
(閉じたら30日は再表示しない)。

### 3-3. アプリバッジ(おまけ・対応環境のみ)

インストール済みPWAで `navigator.setAppBadge()` により
復習待ち件数をアイコンにバッジ表示。ページを開いた時にしか更新できない制約は
あるが、実装数行なので入れておいて損はない。

## Phase 4 — 飽きさせない(中身の持続性)

### 4-1. 復習の間隔反復(かんたんSRS)

現状は間違えた問題が常時復習対象。忘却曲線に沿って「今日復習すべき問題」を絞る:

- 復習で正解するたび次回出題を 1日後 → 3日後 → 7日後 と延ばし、7日後の正解で克服
- state変更: `wrong[key]` に `due`(次回日付)と `step` を追加(既存データは due=今日 で移行)
- ホームの復習カードを「今日の復習 ◯問」に変え、今日の5問とセットの日課にする

### 4-2. 腕試しモード(エンドレスの受け皿)

全問からランダム10問。全ステージ星3を取った人の日課の受け皿として、
記録タブまたはホーム下部から常時挑戦可能に。ハイスコア(連続正解)を記録。

### 4-3. 問題プールの拡充

習慣が続くかどうかの最終的な鍵はコンテンツ量。各分野のステージ問題プールを
5問→8〜10問に増やし、ステージ挑戦のたびに5問をランダム抽出する方式へ
(CLAUDE.md の難易度定義に従って追加)。

## 実装時の共通注意(CLAUDE.md より)

- `js/data.js` や `js/app.js` の変更時は `sw.js` の `CACHE` バージョンを上げる
- 既存セーブデータとの互換: 新フィールドは `defaultState()` に追加し、
  `loadState()` のマージで補完する既存パターンに従う
- 既存問題の並べ替えは localStorage キー(`catId:stageIdx:qIdx`)とずれるため行わない
