# サブ画面 UX 改善プラン(subscreens-ux)

> **進捗**: Phase A〜C まですべて実装済み。実測はライブラリ 新規553px /
> 進行中612px、復習 待ち6問で633px / 空状態258px、記録 進行中約1,040px。
> いずれも概ね1画面(復習・ライブラリ)〜1画面+α(記録)に収まった。

> 対象:**ライブラリ / 復習 / 記録** の3タブ。
> 課題意識:ホーム(home-ux-plan)は3ゾーン化で1画面に収まったが、
> サブ画面は「全件を縦に並べる」構成のまま残っており、特にライブラリは
> 1分野で約3画面分スクロールする。ホームで確立した型(行動はリスト行・
> 説明文は持たせない・詳細はタップ展開)を3画面に横展開する。

## 現状の概算(CSS実測値からの計算。進行中ユーザー想定)

| 画面 | 構成 | 縦の概算 | 主因 |
|---|---|---|---|
| ライブラリ | 説明ノート3行+場面ピル+分野タブ+**1分野24行**(未解放は「???」行) | 約1,400px(約3画面) | 未解放カードも1行ずつ占有。24行は常に全表示 |
| 復習 | CTAカード+分野別内訳カード+苦手問題カード(最大8件×2行) | 約900px | 3カードに分散。毎回同じ説明文。待ち0でも器が残る |
| 記録 | サマリー6タイル+カレンダー+分野別正答率+テスト履歴(+実績29個) | 約1,100px(実績タブは約1,000px) | カレンダーのセルが46px(「n問」内蔵)。履歴・実績が全件表示 |

## 共通方針 — ホームの型の横展開

1. **説明文は行に持たせない。** 定型の案内文は空状態(0件)のときだけ出す
2. **一覧は「見える価値のある行」だけ常時表示。** 未解放・下位・古いものは
   集約行(「残り n件」)かアコーディオンに畳み、タップで展開
3. **状態はテキスト/ピル、行動はリスト行、詳細はタップ展開**、の型を固定
4. 各画面、よくある状態で**1.5画面(約1,200px)以内**を目標

## Phase A — ライブラリ(効果最大)

1分野=3ステージ×8問=24行を常に全表示しているのが最大の要因。
うち未解放分は「??? + ステージ名」の情報量ゼロの行が並ぶだけで、
新規ユーザーでは画面のほぼ全部が「???」になっている。

### A-1. 説明ノートの撤去

冒頭の3行ノート(「クイズで出会った問題が…」)は、カードが1枚もない
ときだけ空状態メッセージとして表示。通常時はサブタイトルの
「集めた知識カード n / 144」で役割が足りている。

### A-2. ステージ別アコーディオン

24行を「初級 / 中級 / 上級」の3セクションに分け、ヘッダーを
`ステージ名 + n/8 + シェブロン` の開閉行にする。

- デフォルト展開は「カードが増える余地のある最初のステージ」1つだけ
  (全解放済みの分野は初級を展開)。ほかは畳んでおく
- 開閉状態はセッション内のみ保持(保存しない)

### A-3. 未解放カードの集約

「???」行を個別に出すのをやめ、各セクション末尾の
**「未解放 n枚 — クイズに正解すると集まります」の1行**に集約する。

- コレクション感(空きスロットが見える楽しさ)はセクションヘッダーの
  `n/8` と集約行の枚数表示で残す。個別スロットの一覧性より、
  「開いてすぐ自分のカードが読める」ことを優先する判断
- 表示だけの変更で `state.seen` には触れないため、既存ユーザーの
  解放状態への影響はない

### A-4. 場面フィルタピルに件数を併記

「会話で使う 12」のように解放済み件数をピルに出す。0件ピルの
空振りタップ(開いたら空状態)を防ぐ。

**効果**:新規ユーザー 約1,400px → 約400px。進行中ユーザーも
「見出し3行+展開中ステージの解放済み行」だけになり約600pxに収まる。

## Phase B — 復習(低コスト)

### B-1. CTAカードの圧縮と説明文の削除

毎回表示している説明文2行(「間違えた問題から最大5問を…」)を削除し、
カードは「復習待ち n問」+主CTAのみに。ヘッダーのサブタイトル
(「復習待ちが n問 あります」)と情報が重複しているので、サブタイトル側は
「正解すると復習リストから消えます」の仕組み説明に差し替える。

### B-2. 分野別内訳をカードからピル行へ(+分野別復習)

「分野別の内訳」カードを廃止し、CTAカード内に「哲学 3 ・ 音楽 2」の
ピル列として吸収。ピルをタップするとその分野だけに絞って復習を開始する
(`startReview` に catId フィルタ引数を追加)。内訳が"表示"から"導線"になる。

### B-3. 苦手リストの折りたたみ

「間違えた回数が多い問題」は上位3件のみ常時表示し、4件以上あるときは
「すべて見る(n件)」のテキストボタンで展開。行は現状の2行構成を維持
(問題文の冒頭は復習の手がかりとして価値があるため)。

### B-4. 空状態の整理

復習待ち0のときは、無効化したボタンと空カードを出さず、
「復習待ちはありません。間違えた問題がここに溜まります」の1文と
「今日の5問へ」のリスト行1つに置き換える。

**効果**:3カード約900px → 1カード+リスト約550px。待ち0のときは約250px。

## Phase C — 記録(磨き込み)

### C-1. 学習カレンダーのセル圧縮

セル内の「n問」テキストを廃止し、濃淡(l0〜l4)だけで表現する
(凡例は既にある)。`min-height: 46px → 34px`。
学習日のセルはタップ可能にし、タップでフッター行に
「7月3日 ・ 12問 解答」と表示する(既存の `cal-today-note` を
選択日表示に拡張。初期表示は今日)。カード約380px → 約260px。

### C-2. サマリーの1段化

6タイル(2段)を4タイル(1段)に:**連続学習 / 学習日数 / 正答率 / 復習待ち**。

- 「累計解答・累計正解」は正答率タイルの下段に「213/287問」として吸収
- 「復習待ち」タイルはタップで復習タブへ遷移する導線にする

### C-3. 実力判定テスト履歴を直近5件+展開

履歴は受験のたび無制限に伸びるため、直近5件+「すべて見る(n件)」に。
ヘッダーの最高評価ピルは現状維持。

### C-4. 実績の並び替えと折りたたみ

解除済みバッジを先頭に並べ、未達成は後ろへ。未達成が多い序盤は
「未達成 n個を見る」で畳んでおく(解除済みが主役、未達成は目標一覧)。
3列グリッドの密度は現状のままでよい。

**効果**:学習記録パネル約1,100px → 約800px。実績パネルは序盤ほぼ1画面に。

## 実装上の注意

- 既存の `render*` 関数と要素IDは維持し、index.html の器の組み替えと
  各 render 内のマークアップ変更で完結させる
- アコーディオン・「すべて見る」の開閉状態は保存しない(セッション内でよい)。
  保存したくなった場合も state ではなく settings 側に置く
- ライブラリの集約・復習の折りたたみは**表示だけの変更**。
  `state.seen` / `state.wrong` のキー形式には触れない
- カレンダーのセルタップは `<button>` 化し、`aria-pressed` で選択状態を示す
- UI文言に絵文字は使わない(› ▾ などの記号は可)。
  css / js / index.html の変更時は sw.js の CACHE を上げる

## 優先順位の根拠

| Phase | コスト | 効果 |
|---|---|---|
| A ライブラリ | 中 | 3画面 → 1画面弱。もっともスクロールが長い画面が解消(本命) |
| B 復習 | 小 | 3カード → 1カード。分野別復習という機能改善も同時に得られる |
| C 記録 | 中 | カレンダー圧縮で1画面+αに。閲覧頻度は低めなので最後でよい |

A・B・C は互いに独立しており、どの順でも実施できる。
効果とコストのバランスから A → B → C を推奨。
